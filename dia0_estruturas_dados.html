Notacao Big O - Definicao Formal

Definicao Matematica

Dizemos que f(n) = O(g(n)) se existem constantes positivas c e n₀ tais que:
f(n) ≤ c · g(n) para todo n ≥ n₀ Em outras palavras: g(n) e um limite superior assintotico para f(n).

O que isso significa?

  • Descreve o pior caso de execucao
  • Ignora constantes e termos menores
  • Foca no comportamento quando n → ∞
  • Permite comparar algoritmos independente de hardware

Por que usar Big O?

  • Hardware muda, complexidade nao
  • Preve escalabilidade do sistema
  • Identifica gargalos antes de implementar
  • Linguagem universal entre desenvolvedores

Exemplo Pratico

Se T(n) = 3n² + 5n + 100, entao T(n) = O(n²)
Justificativa: Para n grande, o termo n² domina. Os coeficientes 3, 5 e 100 sao irrelevantes assintoticamente.

Classes de Complexidade

Classe Nome n=10 n=100 n=1000 Exemplo
O(1) Constante 1 1 1 Acesso por indice
O(log n) Logaritmica 3 7 10 Busca binaria
O(n) Linear 10 100 1000 Percorrer lista
O(n²) Quadratica 100 10.000 1.000.000 Loop aninhado

Como Calcular a Complexidade

Metodologia de Analise

Passo 1: Identifique as operacoes primitivas (atribuicao, comparacao, aritmetica)
Passo 2: Conte quantas vezes cada operacao executa em funcao de n
Passo 3: Some todas as operacoes para obter T(n)
Passo 4: Simplifique mantendo apenas o termo dominante

Regras de Simplificacao

  • Constantes: O(2n) = O(n)
  • Soma: O(n² + n) = O(n²)
  • Produto: O(n) × O(n) = O(n²)
  • Sequencia: O(n) + O(m) = O(n+m)

Hierarquia de Crescimento

O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

Da esquerda para direita: mais eficiente para menos eficiente

Analise de Codigo - Exemplos

O(1) - Tempo Constante

// Operacoes fixas, independente de n int primeiro = array[0]; // 1 op int ultimo = array[n - 1]; // 1 op int soma = primeiro + ultimo; // 1 op // Total: 3 operacoes = O(1)

O(n) - Tempo Linear

int soma = 0; // 1 op for (int i = 0; i < n; i++) // n iteracoes { soma += array[i]; // 1 op por iteracao } // Total: 1 + n = O(n)

O(n²) - Tempo Quadratico

for (int i = 0; i < n; i++) // loop externo: n vezes { for (int j = 0; j < n; j++) // loop interno: n vezes CADA iteracao externa { matriz[i, j] = i * j; // 1 operacao } } // Total: n * n = n² operacoes = O(n²)

Impacto em Games - 60 FPS = 16.6ms por frame

Com n=1000 inimigos: O(n²) = 1.000.000 operacoes. Se cada leva 1μs = 1 segundo por frame! Inviavel.

Arrays: Conceito

Tempo: 00:00 - 01:00

O que é um Array?

Uma coleção ordenada de elementos do mesmo tipo, armazenados em posições contíguas de memória.

10[0]
25[1]
42[2]
8[3]
15[4]

Características

  • Tamanho fixo
  • Acesso por índice: O(1)
  • Índices começam em 0

Sintaxe C#

int[] numeros = new int[5]; int[] valores = { 10, 25, 42, 8, 15 };

List<T>: Conceito

Por que não Array?

Arrays têm tamanho fixo. Quando precisamos adicionar/remover elementos dinamicamente, usamos List<T>.

Array (Fixo)

  • Tamanho definido na criação
  • Não pode crescer/diminuir
  • Bom para dados estáticos

List<T> (Dinâmico)

  • Tamanho cresce automaticamente
  • Add, Remove, Insert
  • Bom para coleções que mudam
using System.Collections.Generic; List<int> numeros = new List<int>(); numeros.Add(10); // Adiciona: [10] numeros.Add(20); // Adiciona: [10, 20] numeros.Remove(10); // Remove: [20]

Stack (Pilha): Conceito

Tempo: 02:00 - 02:45

LIFO = Last In, First Out

O último elemento a entrar é o primeiro a sair. Como uma pilha de pratos!

← TOPO (Push/Pop)

Ação 1
Ação 2
Ação 3
Ação 4

Operações - Todas O(1)!

  • Push: adiciona no topo
  • Pop: remove do topo
  • Peek: ver topo sem remover
  • Count: quantidade

FIFO = First In, First Out

O primeiro a entrar é o primeiro a sair. Como uma fila de banco!

FRENTE (sai) ← → TRÁS (entra)

2º

Operações - Todas O(1)!

Queue: No Dungeon Of Algorithms

Sistema de Spawn de Inimigos

Vamos criar um sistema que spawna inimigos gradualmente!

// Nova classe: SpawnManager.cs public class SpawnManager { private Queue<string> _spawnQueue = new(); public void ScheduleWave() { _spawnQueue.Enqueue("Slime"); _spawnQueue.Enqueue("Slime"); _spawnQueue.Enqueue("Ghost"); } public string SpawnNext() { if (_spawnQueue.Count > 0) return _spawnQueue.Dequeue(); return null; } }
